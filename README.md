### Алгоритм решения



######



#### Алгоритм начинается с инициализации внутреннего словаря со справочником кодов ошибок и программного создания текстового файла с исходным кодом Паскаля. Далее в память принудительно записываются тестовые ошибки с указанием их кодов, строк и точных позиций символов. На основе этих данных запускается процесс построчного чтения файла, при котором первая строка загружается в буфер, а внутренний метод начинает посимвольно перемещать указатель по каждому знаку. Как только индекс доходит до конца строки, вызывается метод вывода, который печатает текст строки с ее порядковым номером и проверяет наличие ошибок для нее. При обнаружении сбоя программа рассчитывает отступ в пробелах, позиционирует стрелочку-указатель точно под проблемным символом, извлекает описание ошибки из таблицы и увеличивает счетчик. В завершение, после достижения конца файла, вызывается финальный метод, который закрывает файловый поток и выводит итоговое количество зафиксированных ошибок на экран.
